# Showcase Screenshots

The public screenshot capture script should display more than the default reader path. The expanded set is intended to show the app as a full study workspace rather than only a static Bible reader.

Generated by:

```powershell
npm run screenshots:public
```

Expected screenshots:

| File | Purpose |
|---|---|
| `docs/images/reader.png` | Main reader view. |
| `docs/images/book-picker.png` | Book picker / Old and New Testament navigation surface. |
| `docs/images/detail-panel.png` | Outline/detail side panel. |
| `docs/images/verse-context-tabs.png` | Verse action tabs and cross-reference workflow. |
| `docs/images/interlinear.png` | Interlinear word-by-word side panel. |
| `docs/images/hebrew-side-panel.png` | Reader-to-Hebrew / Strong's detail logic. |
| `docs/images/search.png` | Local search workflow. |
| `docs/images/study-marks.png` | Study Marks / tagged target dashboard. |
| `docs/images/study-data.png` | Study Data export/import and local storage summary. |
| `docs/images/local-processing.png` | Local Processing / study job status. |
| `docs/images/mobile.png` | Mobile reader layout. |

Only update README image references after the new screenshots are generated and visually reviewed. The README should not point to missing images.
